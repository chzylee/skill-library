# The Recording Standard (enforcement copy)

Derived: 2026-07-14 from the canonical page — https://app.notion.com/p/39276356d6fe8139b351fff7cb417e99
Do not hand-edit. When the standard changes, re-copy from the enforcement-copy subpage there.

Every written thing has one home you can name without deciding. The rule:
CLASSIFY what it is (one of the 5 page modes, or a known type) -> route to its
deterministic home -> if no home exists, ASK (then register it). Never improvise
a scattered destination; precision beats speed. Duplication only as pointers.

The 5 page modes (classify by read/write contract, not topic):
1 Reference | 2 Living documentation | 3 Operational surface | 4 Standard/contract | 5 Log/archive.
Every page declares its mode in its opener.

Routes you cannot derive (state these; everything else co-locates with its subject):
- Work executed (any session) -> ts-pmo Work Log, via debrief.
- Durable finding / lesson -> the page where it changes behavior; personal lessons -> auto-memory.
- Session work done with Claude Code -> the filesystem session stream (~/.claude/projects/**/*.jsonl).
- Anything with no better home -> the miscellaneous home: Claude Projects, routed to its bucket.
- Position -> system: Up Next; per-build: repo build_status.md; career: OPEN. Wikis never track position.

Derivable routes (principles, not rows):
- Co-locate with the subject: an audit/experiment/domain-fact -> the space of the thing it's about.
  A system self-audit -> Reviews and Changes; a standalone large analysis -> Agentic Analyses.
- Current state vs history vs position: the doc that IS the thing shows current state only;
  history -> a Changelog; position -> its one position home.
- Build code + run-state -> the repo (README, decision_log.md, TEST_SPEC.md, RATIFICATION_LOG.md,
  build_status.md); the wiki only points to it.
- Skills route by kind (module repo / claude-notion-flywheel / own repo); the "library shows every
  skill" invariant lives on the skill-library page.

Recording channels a consumer reads: git history | Notion updates | the filesystem session stream.
